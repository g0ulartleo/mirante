# Mirante Alarms Runtime

Generated by `mirante init repo`.

This repository owns alarm definitions and implementations.
Mirante core syncs alarm metadata from this server application and uses it to execute alarms.

## local runtime

```sh
npm install
npm run dev
```

the runtime listens on `ALARM_RUNTIME_ADDR`. default local address is `127.0.0.1:50051`.

## alarm files

alarm files live in `src/alarms`. exported alarm objects should use `AlarmDefinition` from `@mirante/alarms`.

`AlarmDefinition` is the source alarm object you write. `Alarm` is the runtime metadata shape returned over gRPC.

```ts
import { healthy, type AlarmDefinition } from "@mirante/alarms";

export const checkServerCount: AlarmDefinition = {
  id: "check-server-count",
  description: "Describe what this alarm checks.",
  interval: "1m",
  notifications: {
    critical: {
      slackWebhooks: [process.env.CRITICAL_SLACK_WEBHOOK_URL!],
    },
    warnings: {
      slackWebhooks: [process.env.WARNING_SLACK_WEBHOOK_URL!],
    },
  },
  async run() {
    return healthy("OK");
  },
};
```

notification channels must be grouped by severity: `critical` and `warnings`.

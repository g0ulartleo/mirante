# Mirante Alarms Runtime

Generated by `mirante init repo`.

This repository owns alarm definitions and implementations.
Mirante core syncs alarm metadata from this server application and uses it to execute alarms.

## Local Runtime

```sh
go mod tidy
go run ./cmd/runtime
```

The runtime listens on `ALARM_RUNTIME_ADDR`. Default local address is `127.0.0.1:50051`.

## Docker Compose

```sh
cp .env.example .env
docker compose up --build
```

Docker Compose reads `.env`; local `go run` uses your shell environment.

Services:

- Dashboard/API: http://localhost:40169
- Alarm runtime: localhost:50051
- Redis: localhost:6379

## Add Alarms

```sh
mirante new alarm server-events-dlq
```

Generated alarm files are written to `internal/alarms`. Register new alarms in `cmd/runtime/main.go` by adding them to the `Alarms` list.
